# AI Code Review Skills

懂业务的代码审查 Skill —— 先告诉你"改了什么业务"，再告诉你"代码哪里有问题"。

## 特色

不同于传统工具只告诉你"代码哪里写错了"，我们先用自然语言告诉你"这次改了什么业务"——移除了哪个按钮、新增了哪个接口、调整了哪段逻辑——让 reviewer 快速理解变更意图，再有针对性地审查代码。

## 包含三个 Skill

| Skill | 适用场景 | 说明 |
|-------|----------|------|
| **code-review** | 前后端混合项目 | 自动识别前后端，按需应用规则 |
| **frontend-code-review** | 纯前端项目 | React/Vue/Angular/Next.js 等，60+ 条前端专属规则 |
| **backend-code-review** | 纯后端项目 | Python/Java/Go/Node.js 等，60+ 条后端专属规则 |

## 核心特性

### 业务级变更概述

```
## 本次改了什么

- [新增] 用户注册接口，支持邮箱验证码校验
- [修改] 用户列表页移除"删除用户"按钮及相关确认弹窗
- [修复] 修复并发场景下优惠券重复使用的问题
```

用自然语言告诉 reviewer 这次改了什么业务功能，不用翻 diff 就能判断变更是否合理。

### 技术栈自动适配

首次使用自动检测项目技术栈（React/Vue/Django/Spring Boot 等），生成专属审查配置模板，支持自定义规则。

### 三层信息架构

```
┌─────────────────────────────────────┐
│  本次改了什么（业务概述）              │ ← PM/项目经理一眼看懂
├─────────────────────────────────────┤
│  技术变更摘要（文件/行数统计）        │ ← 技术负责人看范围
├─────────────────────────────────────┤
│  具体问题 P0-P3（diff + 修复建议）    │ ← 开发者看细节
└─────────────────────────────────────┘
```

### Git 变更级精准审查

默认只审查本次 Git 变更，一句 "review 一下" 即可生成结构化报告。

## 检查类别

### 通用规则（前后端都适用）
- 代码质量：命名、类型安全、职责单一
- 安全性：注入攻击、认证授权、敏感数据
- 错误处理：异常捕获、日志记录、降级策略

### 前端专属规则
- 性能：重渲染、内存泄漏、代码分割
- 可访问性：语义化 HTML、ARIA、键盘导航
- 样式：CSS 规范、响应式、设计 token

### 后端专属规则
- 性能：数据库查询、缓存、并发处理
- API 设计：RESTful 规范、版本控制、幂等性
- 数据库：索引、事务、数据一致性

## 安装

将 skill 目录复制到项目对应位置：

| 编程助手 | 放置路径 |
|----------|----------|
| **Trae** | `.trae/skills/<skill-name>/` |
| **Claude Code** | `.claude/skills/<skill-name>/` |
| **Cursor** | `.cursor/skills/<skill-name>/` |
| **Gemini CLI** | `.gemini/skills/<skill-name>/` |

例如安装通用版 code-review：

```bash
# Trae
cp -r code-review/ your-project/.trae/skills/

# Claude Code
cp -r code-review/ your-project/.claude/skills/
```

## 使用

对 AI 助手说：

- `"review 一下"` → 审查本次 Git 变更（默认）
- `"审查整个项目"` → 全量审查
- `"审查 src/api/user.py"` → 文件定向审查

## 兼容

支持所有主流 AI 编程助手：Claude Code、Cursor、Gemini CLI、Trae 等。
